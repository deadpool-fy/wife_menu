# 云开发环境配置指南

## 问题现象

云函数提示"未选择环境"，说明云开发环境未正确配置。

## 解决步骤

### 第一步：检查云开发环境

1. **在微信开发者工具中**：
   - 点击 **"云开发"** 按钮
   - 查看是否显示环境信息

2. **如果没有环境**：
   - 点击 **"开通"** 或 **"创建环境"**
   - 选择环境名称（如：`wife-menu-env`）
   - 等待创建完成

### 第二步：获取环境ID

1. **在云开发控制台**：
   - 点击 **"设置"** → **"环境设置"**
   - 复制 **"环境ID"**（格式如：`cloud1-xxxxxxxxx`）

2. **更新 app.js**：
   - 将环境ID替换到 `app.js` 中的 `env` 字段

### 第三步：配置云函数环境

1. **在云函数列表中**：
   - 右键 `sendMessage` 函数
   - 选择 **"配置"**
   - 确认环境ID正确

2. **重新部署云函数**：
   - 右键 `sendMessage` 函数
   - 选择 **"上传并部署"**

### 第四步：验证配置

1. **测试云函数**：
   - 右键 `sendMessage` → **"本地调试"**
   - 输入测试参数
   - 查看执行结果

2. **检查权限**：
   - 在云开发控制台
   - **"设置"** → **"权限设置"**
   - 确认 **"订阅消息"** 权限已开启

## 详细操作步骤

### 1. 开通云开发

1. 打开微信开发者工具
2. 点击工具栏的 **"云开发"** 按钮
3. 如果显示"未开通"，点击 **"开通"**
4. 选择 **"创建环境"**
5. 输入环境名称：`wife-menu-env`
6. 选择 **"基础版"**（免费）
7. 点击 **"确定"**

### 2. 获取环境ID

1. 云开发开通后，点击 **"设置"**
2. 在 **"环境设置"** 中查看 **"环境ID"**
3. 复制环境ID（如：`cloud1-5ga4h58zc0ea35dc`）

### 3. 更新配置文件

**更新 app.js**：
```javascript
wx.cloud.init({
  env: 'cloud1-5ga4h58zc0ea35dc', // 替换为您的环境ID
  traceUser: true
})
```

**更新 project.config.json**：
```json
{
  "cloudfunctionRoot": "cloudfunctions/",
  "cloudfunctionTemplateRoot": "cloudfunctionTemplate/"
}
```

### 4. 部署云函数

1. 右键 `cloudfunctions/sendMessage` 文件夹
2. 选择 **"上传并部署"**
3. 等待部署完成
4. 查看部署状态是否为 **"正常"**

### 5. 配置权限

1. 在云开发控制台
2. 点击 **"设置"** → **"权限设置"**
3. 找到 **"订阅消息"** 权限
4. 点击 **"开启"**

## 常见问题

### Q1: 环境ID在哪里查看？
A: 云开发控制台 → 设置 → 环境设置 → 环境ID

### Q2: 云函数部署失败怎么办？
A: 
1. 检查环境ID是否正确
2. 确认云开发已开通
3. 重新部署云函数

### Q3: 权限设置在哪里？
A: 云开发控制台 → 设置 → 权限设置

### Q4: 如何测试云函数？
A: 
1. 右键云函数 → 本地调试
2. 输入测试参数
3. 查看执行结果
