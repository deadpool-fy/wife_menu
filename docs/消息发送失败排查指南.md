# 消息发送失败排查指南

## 问题分析

根据控制台日志，云函数调用成功，但模板消息发送失败。可能的原因包括：

### 1. 模板ID问题
- 模板ID不正确
- 模板未审核通过
- 模板字段不匹配

### 2. 用户授权问题
- 用户未授权订阅消息
- 授权已过期
- 用户拒绝接收消息

### 3. 云函数权限问题
- 云函数没有订阅消息权限
- 环境配置错误

### 4. 参数问题
- 模板字段数据格式错误
- 必填字段缺失

## 排查步骤

### 第一步：检查云函数日志

1. 在微信开发者工具中：
   - 点击 **"云开发"** → **"云函数"**
   - 找到 `sendMessage` 函数
   - 点击 **"日志"** 查看详细错误信息

2. 重新测试并查看日志：
   - 选择菜品 → 点击"发送给宝宝"
   - 授权订阅消息
   - 查看云函数日志输出

### 第二步：验证模板ID

1. 在微信公众平台：
   - 登录 https://mp.weixin.qq.com/
   - 进入 **"功能"** → **"订阅消息"**
   - 查看模板状态是否为 **"审核通过"**

2. 检查模板字段：
   - 确认模板包含 `thing1`、`thing2`、`time3` 字段
   - 如果字段不匹配，需要修改云函数代码

### 第三步：检查用户授权

1. 清除小程序数据：
   - 微信 → 发现 → 小程序 → 家庭菜单
   - 右上角设置 → 清除数据

2. 重新测试授权流程：
   - 进入小程序
   - 选择菜品 → 点击"发送给宝宝"
   - 确认授权弹窗出现
   - 点击"去授权" → 选择"总是"

### 第四步：检查云函数权限

1. 在云开发控制台：
   - 进入 **"设置"** → **"权限设置"**
   - 确认 **"订阅消息"** 权限已开启

2. 检查云函数配置：
   - 确认云函数已正确部署
   - 检查环境ID是否正确

## 常见错误码

### 40001 - 模板ID无效
**原因：** 模板ID不存在或未审核通过
**解决：** 检查模板ID，确认模板状态

### 40003 - 用户未授权
**原因：** 用户未授权订阅消息
**解决：** 重新请求用户授权

### 40004 - 模板字段不匹配
**原因：** 发送的数据字段与模板不匹配
**解决：** 修改云函数中的字段名称

### 40005 - 发送频率限制
**原因：** 用户每月接收消息次数超限
**解决：** 等待下月重置或使用其他发送方式

## 修复方案

### 方案1：修改模板字段

如果模板字段不匹配，修改云函数代码：

```javascript
// 根据实际模板字段修改
const templateData = {
  touser: targetUserId,
  template_id: templateId,
  page: 'pages/index/index',
  data: {
    // 根据实际模板字段调整
    keyword1: { value: '家庭菜单小程序' },
    keyword2: { value: message },
    keyword3: { value: new Date().toLocaleString('zh-CN') }
  },
  miniprogram_state: 'formal'
}
```

### 方案2：添加错误处理

在云函数中添加更详细的错误处理：

```javascript
try {
  const result = await cloud.openapi.subscribeMessage.send(templateData)
  
  if (result.errCode === 0) {
    return { success: true, message: '发送成功' }
  } else {
    // 记录具体错误信息
    console.log('订阅消息发送失败:', {
      errCode: result.errCode,
      errMsg: result.errMsg,
      templateData: templateData
    })
    
    return {
      success: false,
      message: `发送失败: ${result.errMsg} (${result.errCode})`
    }
  }
} catch (error) {
  console.error('发送异常:', error)
  return {
    success: false,
    message: `发送异常: ${error.message}`
  }
}
```

### 方案3：降级处理

如果订阅消息持续失败，可以添加降级处理：

```javascript
// 在 messageService.js 中
function sendTemplateMessage(message, resolve, reject) {
  auth.requestSubscribeMessage('your_template_id')
    .then(() => {
      sendTemplateMessageToServer(message)
        .then(resolve)
        .catch((error) => {
          console.log('订阅消息发送失败，降级到复制功能:', error)
          // 降级到复制功能
          fallbackToClipboard(message, resolve, reject)
        })
    })
    .catch((error) => {
      console.log('用户拒绝授权，降级到复制功能:', error)
      fallbackToClipboard(message, resolve, reject)
    })
}
```
