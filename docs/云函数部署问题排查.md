# 云函数部署问题排查

## 问题现象

云函数调用成功，但返回的是输入参数而不是预期的结果对象，说明云函数没有正确执行。

## 可能原因

### 1. 云函数未正确部署
- 代码修改后没有重新部署
- 部署过程中出现错误
- 云函数版本不匹配

### 2. 环境配置问题
- 云开发环境ID不正确
- 云函数权限配置错误
- 订阅消息权限未开启

### 3. 代码执行问题
- 云函数代码有语法错误
- 依赖包缺失
- 执行超时

## 排查步骤

### 第一步：检查云函数部署状态

1. **在微信开发者工具中**：
   - 点击 **"云开发"** → **"云函数"**
   - 找到 `sendMessage` 函数
   - 查看状态是否为 **"正常"**

2. **重新部署云函数**：
   - 右键 `cloudfunctions/sendMessage` 文件夹
   - 选择 **"上传并部署"**
   - 等待部署完成

### 第二步：检查云函数日志

1. **查看部署日志**：
   - 在云函数列表中点击 `sendMessage`
   - 查看 **"部署日志"** 是否有错误

2. **查看运行日志**：
   - 点击 **"日志"** 标签
   - 重新测试消息发送
   - 查看是否有新的日志输出

### 第三步：验证云函数执行

1. **本地调试**：
   - 右键 `cloudfunctions/sendMessage` → **"本地调试"**
   - 输入测试参数：
     ```json
     {
       "message": "测试消息",
       "targetUserId": "Dingding7654321_",
       "templateId": "tLEKQiiMe8JDjm1GIGq5UDbHrZNZX0bxOhuRM0zho4g"
     }
     ```
   - 查看执行结果

2. **检查返回结果**：
   - 确认返回的是 `{success: true/false, message: "..."}` 格式
   - 而不是输入参数

### 第四步：检查权限配置

1. **云开发权限**：
   - 进入云开发控制台
   - **"设置"** → **"权限设置"**
   - 确认 **"订阅消息"** 权限已开启

2. **云函数权限**：
   - 在云函数列表中查看权限状态
   - 确认有调用微信API的权限

## 修复方案

### 方案1：重新部署云函数

1. **删除旧版本**：
   - 在云开发控制台删除 `sendMessage` 函数
   - 重新上传部署

2. **检查依赖**：
   - 确认 `package.json` 包含 `wx-server-sdk`
   - 版本为 `~2.6.3`

### 方案2：简化云函数代码

如果问题持续，可以先用简化版本测试：

```javascript
// cloudfunctions/sendMessage/index.js
const cloud = require('wx-server-sdk')

cloud.init({
  env: cloud.DYNAMIC_CURRENT_ENV
})

exports.main = async (event, context) => {
  console.log('云函数执行，参数:', event)
  
  // 简单返回测试
  return {
    success: true,
    message: '云函数执行成功',
    receivedData: event
  }
}
```

### 方案3：检查环境配置

1. **确认环境ID**：
   - 在 `app.js` 中确认环境ID正确
   - 与云开发控制台中的环境ID一致

2. **重新初始化**：
   - 删除云开发配置
   - 重新创建云开发环境
