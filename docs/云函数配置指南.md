# 微信小程序云函数配置指南

## 云函数部署步骤

### 1. 开通云开发

#### 在微信开发者工具中：
1. 打开微信开发者工具
2. 点击 **"云开发"** 按钮
3. 开通云开发服务
4. 选择环境（推荐：生产环境）
5. 记录环境ID（如：`env-xxxxx`）

### 2. 创建云函数

#### 在微信开发者工具中：
1. 右键点击 `cloudfunctions` 文件夹
2. 选择 **"新建云函数"**
3. 输入函数名：`sendMessage`
4. 选择运行环境：Node.js

### 3. 上传云函数

#### 方法一：使用微信开发者工具
1. 右键点击 `cloudfunctions/sendMessage` 文件夹
2. 选择 **"上传并部署：云端安装依赖"**
3. 等待上传完成

#### 方法二：使用命令行
```bash
# 进入云函数目录
cd cloudfunctions/sendMessage

# 安装依赖
npm install

# 上传云函数（需要在微信开发者工具中执行）
```

### 4. 配置环境ID

#### 修改 app.js
```javascript
wx.cloud.init({
  env: 'your-env-id', // 替换为实际环境ID
  traceUser: true
})
```

#### 修改 utils/messageService.js
```javascript
// 替换模板ID
templateId: 'xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx' // 替换为实际模板ID
```

## 云函数配置

### 1. 云函数权限设置

#### 在云开发控制台：
1. 进入 **"云函数"** 页面
2. 找到 `sendMessage` 函数
3. 点击 **"权限设置"**
4. 设置为 **"所有用户可调用"**

### 2. 订阅消息权限

#### 在云开发控制台：
1. 进入 **"设置"** → **"权限设置"**
2. 开启 **"订阅消息"** 权限
3. 配置相关权限

## 测试云函数

### 1. 本地测试

#### 在微信开发者工具中：
1. 右键点击 `sendMessage` 云函数
2. 选择 **"本地调试"**
3. 输入测试参数：
```json
{
  "message": "测试消息",
  "targetUserId": "Dingding7654321_",
  "templateId": "your_template_id"
}
```

### 2. 真机测试

#### 在小程序中：
1. 选择菜品
2. 点击"发送给宝宝"
3. 授权订阅消息
4. 查看是否发送成功

## 云函数日志

### 查看日志
1. 在云开发控制台
2. 进入 **"云函数"** → **"sendMessage"**
3. 点击 **"日志"** 查看运行日志

### 常见错误
- **权限不足**：检查云函数权限设置
- **模板ID错误**：确认模板ID正确
- **用户未授权**：确认用户已授权订阅消息

## 故障排除

### 1. 云函数调用失败
- 检查环境ID是否正确
- 确认云函数已部署
- 查看云函数日志

### 2. 订阅消息发送失败
- 检查模板ID是否正确
- 确认用户已授权
- 查看错误码对照表

### 3. 权限问题
- 检查云函数权限设置
- 确认订阅消息权限已开启
- 验证用户身份

## 高级配置

### 1. 环境变量
```javascript
// 在云函数中
const templateId = process.env.TEMPLATE_ID
const targetUserId = process.env.TARGET_USER_ID
```

### 2. 数据库集成
```javascript
// 保存发送记录
const db = cloud.database()
await db.collection('messages').add({
  data: {
    message: message,
    targetUserId: targetUserId,
    sendTime: new Date(),
    status: 'success'
  }
})
```

### 3. 错误重试
```javascript
// 重试机制
let retryCount = 0
const maxRetries = 3

while (retryCount < maxRetries) {
  try {
    const result = await cloud.openapi.subscribeMessage.send(templateData)
    if (result.errCode === 0) break
  } catch (error) {
    retryCount++
    if (retryCount >= maxRetries) throw error
    await new Promise(resolve => setTimeout(resolve, 1000))
  }
}
```
