# 云函数执行问题排查

## 问题现象

云函数调用成功，但返回的是入參 `event` 对象，而不是云函数 `main` 方法的返回值。说明云函数的 `main` 方法没有被正确执行。

## 问题分析

### 日志分析
```javascript
// 云函数调用成功
{errMsg: "cloud.callFunction:ok", result: {...}, requestID: "..."}

// 但 result 是入參 event
result: {
  message: "今天中午的菜单：…",
  targetUserId: "Dingding7654321_",
  templateId: "tLEKQiiMe8JDjm1GIGq5UDbHrZNZX0bxOhuRM0zho4g",
  userInfo: {...}
}

// 而不是期望的返回值
{
  success: true/false,
  message: "...",
  data: ...
}
```

### 可能原因
1. **云函数 main 方法未执行**
2. **云函数代码有语法错误**
3. **云函数未正确部署**
4. **环境配置问题**

## 排查步骤

### 第一步：验证云函数是否执行

已在 `cloudfunctions/sendMessage/index.js` 中添加测试返回：

```javascript
exports.main = async (event, context) => {
  console.log('=== 云函数开始执行 ===')
  
  // 立即返回测试
  return {
    success: true,
    message: '云函数正常执行',
    event: event,
    timestamp: new Date().toISOString()
  }
  
  // 原始代码...
}
```

### 第二步：重新部署云函数

1. **重新部署**：
   - 右键 `cloudfunctions/sendMessage` → **"上传并部署"**
   - 等待部署完成

2. **确认部署成功**：
   - 在云开发控制台查看函数状态
   - 确认状态为 **"正常"**

### 第三步：测试云函数

1. **本地调试**：
   - 右键 `sendMessage` → **"本地调试"**
   - 输入测试参数：
     ```json
     {
       "message": "测试消息",
       "targetUserId": "Dingding7654321_",
       "templateId": "tLEKQiiMe8JDjm1GIGq5UDbHrZNZX0bxOhuRM0zho4g"
     }
     ```
   - 查看执行结果

2. **小程序测试**：
   - 在小程序中测试消息发送
   - 查看返回结果

### 第四步：检查云函数日志

1. **在云开发控制台**：
   - 点击 `sendMessage` 函数
   - 查看 **"日志"** 标签
   - 重新测试，查看是否有新日志

2. **期望的日志输出**：
   ```
   === 云函数开始执行 ===
   接收到的参数: {...}
   微信上下文: {...}
   === 测试返回 ===
   ```

## 修复方案

### 方案1：简化云函数代码

如果问题持续，使用最简版本：

```javascript
// cloudfunctions/sendMessage/index.js
const cloud = require('wx-server-sdk')

cloud.init({
  env: cloud.DYNAMIC_CURRENT_ENV
})

exports.main = async (event, context) => {
  console.log('云函数执行，参数:', event)
  
  return {
    success: true,
    message: '测试成功',
    receivedParams: event
  }
}
```

### 方案2：检查语法错误

1. **语法检查**：
   - 在开发者工具中查看是否有语法错误
   - 检查 `package.json` 配置

2. **重新初始化**：
   - 删除云函数
   - 重新创建部署

### 方案3：环境重新配置

1. **删除云函数**：
   - 在云开发控制台删除 `sendMessage`
   - 重新部署

2. **检查环境设置**：
   - 确认环境ID正确
   - 重新初始化云开发
