# 直接发送微信消息配置指南

## 功能说明

现在小程序支持直接通过微信模板消息发送给指定用户，无需手动复制粘贴。

## 实现方案

### 1. 微信模板消息
- **优点**：直接发送到用户微信
- **缺点**：需要用户授权，有发送限制

### 2. 客服消息
- **优点**：无需用户授权
- **缺点**：需要企业微信配置

### 3. 降级方案
- **复制到剪贴板**：当其他方式失败时自动降级

## 部署步骤

### 第一步：部署后端服务

1. **上传服务器代码**
   ```bash
   # 将 server 文件夹上传到云服务器
   cd server
   npm install
   npm start
   ```

2. **配置微信小程序**
   - 登录微信公众平台：https://mp.weixin.qq.com/
   - 获取 AppSecret：开发 → 开发设置 → 开发者ID
   - 申请模板消息：功能 → 模板消息 → 添加模板

3. **修改服务器配置**
   ```javascript
   // server/wechat-message-sender.js
   const WECHAT_CONFIG = {
     appId: 'wx5783986edcbaba40',
     appSecret: 'your_app_secret', // 替换为实际AppSecret
     templateId: 'your_template_id', // 替换为实际模板ID
     targetUserId: 'Dingding7654321_'
   };
   ```

### 第二步：配置小程序

1. **修改服务器地址**
   ```javascript
   // utils/messageService.js
   wx.request({
     url: 'https://your-server.com/api/send-template-message', // 替换为实际服务器地址
     // ...
   });
   ```

2. **配置服务器域名**
   - 在微信公众平台添加服务器域名到白名单
   - 开发 → 开发设置 → 服务器域名

## 使用流程

### 发送菜单
1. 选择菜品
2. 点击"发送给宝宝"
3. 系统尝试发送模板消息
4. 如果成功：显示"消息已直接发送给宝宝"
5. 如果失败：自动降级到复制功能

### 发送评价
1. 完成评分
2. 点击"发送给宝宝"
3. 系统尝试发送模板消息
4. 如果成功：显示"消息已直接发送给宝宝"
5. 如果失败：自动降级到复制功能

## 注意事项

1. **安全性**
   - 不要将 AppSecret 提交到代码仓库
   - 使用环境变量存储敏感信息

2. **模板消息限制**
   - 需要用户主动授权
   - 有发送频率限制
   - 用户可随时取消授权

3. **服务器配置**
   - 确保服务器域名已添加到微信白名单
   - 使用 HTTPS 协议
   - 配置适当的 CORS 头

## 故障排除

### 1. 模板消息发送失败
- 检查用户是否已授权
- 确认模板ID是否正确
- 查看微信错误码

### 2. 服务器请求失败
- 检查服务器是否正常运行
- 确认域名配置是否正确
- 查看网络连接

### 3. 用户未收到消息
- 确认用户微信ID是否正确
- 检查用户是否已关注小程序
- 确认用户授权状态

## 测试建议

1. **本地测试**
   - 使用 ngrok 等工具暴露本地服务

2. **生产测试**
